# 右上角实时指数行情设计

## 背景

当前首页右上角行情组件展示上证、深证、恒生和标普指数，但所有数值均由浏览器中的随机数生成。它们带有 `ONLINE` 和 `SYNC` 等视觉标签，容易被误认为真实行情。

本次改造将完全删除模拟行情，新增一个独立 Cloudflare Worker，从免密钥的腾讯公开行情端点获取五个指数的最新报价、标准化为 JSON 并缓存 20 秒。网站只访问该 Worker，不直接依赖第三方响应格式。该数据用于个人站点的信息展示，不作为交易级数据源，也不承诺无延迟或长期可用。

## 目标

- 展示纳斯达克综合指数、标普 500、沪深 300、科创 50、中证 500。
- 页面打开后立即请求一次，页面可见时每 30 秒更新一次。
- 通过独立 Worker 代理和 20 秒边缘缓存控制上游请求频率。
- 展示报价时间、交易中或闭市状态以及数据是否陈旧。
- 数据源异常时保留最后一次成功结果，不制造或猜测价格。
- 将数据获取和解析隔离，便于将来切换为自建代理或正式 API。
- 通过单元测试覆盖解析、异常响应、时效状态和刷新策略。

## 非目标

- 不提供分时图、K 线、成交明细或交易功能。
- 不引入需要密钥的行情服务。
- 不迁移现有 Astro 网站到 Cloudflare，也不更改 `finalwhite.lol` 的域名和 OpenResty 托管方式。
- 不引入数据库、KV、D1、队列或常驻服务器进程。
- 不保证交易所级实时性，也不用于投资决策。
- 不实现完整的全球交易所节假日日历。

## 数据源与代码映射

请求端点：

```text
https://qt.gtimg.cn/q=sh000300,sh000688,sh000905,usIXIC,usINX
```

映射如下：

| 页面代码 | 数据代码 | 名称 |
|---|---|---|
| `NASDAQ` | `usIXIC` | 纳斯达克综合指数 |
| `SP500` | `usINX` | 标普 500 |
| `CSI300` | `sh000300` | 沪深 300 |
| `STAR50` | `sh000688` | 科创 50 |
| `CSI500` | `sh000905` | 中证 500 |

接口返回 GBK 编码的多行赋值文本。每行使用 `~` 分隔字段；本实现只读取固定代码、最新点位、涨跌额、涨跌幅和行情时间。返回内容必须作为普通文本解析，禁止使用 `eval` 或动态执行。

腾讯端点不是具有稳定性承诺的正式开放 API，因此端点地址、字段映射和解析逻辑必须集中在 Worker 内部。浏览器不读取原始 GBK 响应。

## 架构

### 独立 Cloudflare Worker

Worker 位于 `workers/market-feed/`，使用独立的 `wrangler.jsonc`，不复用仓库根目录的旧 Cloudflare 配置。

- Worker 名称固定为 `dongfangbai-market-feed`。
- Worker 通过默认 `workers.dev` 地址发布，不绑定 `finalwhite.lol` 路由。
- 唯一业务接口为 `GET /markets`。
- 其他路径返回 JSON 404，非 `GET`/`OPTIONS` 方法返回 405。
- `OPTIONS /markets` 返回预检响应。
- CORS 只允许 `https://finalwhite.lol`、`https://www.finalwhite.lol`、`http://localhost:4321` 和 `http://127.0.0.1:4321`。

Worker 先使用规范化的 `/markets` 请求作为 `caches.default` 缓存键。命中时直接返回；未命中时请求腾讯行情、解析并生成 JSON，设置 `Cache-Control: public, max-age=20` 后写入当前边缘数据中心的缓存。缓存是性能优化而非持久存储。

Cloudflare Workers 的标准 `TextDecoder` 只保证 UTF-8。腾讯响应中的代码、分隔符、数字和时间均为 ASCII，因此 Worker 读取 `ArrayBuffer` 后逐字节保留 ASCII 字节、将非 ASCII 名称字节替换为空格，再按变量名和 `~` 分隔符解析。页面名称使用本地白名单映射，不依赖 GBK 解码，也不引入额外编码库。

Worker 的 JSON 响应结构为：

```ts
interface MarketFeedResponse {
  source: 'Tencent';
  fetchedAt: string;
  quotes: MarketQuotePayload[];
}

interface MarketQuotePayload {
  id: 'NASDAQ' | 'SP500' | 'CSI300' | 'STAR50' | 'CSI500';
  value: number;
  change: number;
  percent: number;
  quotedAt: string;
}
```

### 行情组件

`MarketHud.astro` 只负责静态结构和初始状态：

- 五行指数均以 `--` 作为初始点位和涨跌幅。
- 每行通过稳定的 `data-market` 属性标识指数。
- 底部状态初始显示 `SYNCING`，并预留来源和行情时间位置。
- 页面中不保留任何看似真实的硬编码行情数值。

### 浏览器行情模块

新增独立的客户端模块，职责包括：

1. 从独立配置读取部署后确定的 Worker `/markets` 地址。
2. 使用 `fetch` 和 5 秒超时获取标准 JSON。
3. 验证响应结构、白名单代码和有限数值。
4. 将标准化结果交给 DOM 更新函数。
5. 管理刷新、可见性和失败退避。

浏览器内部使用的单条行情结构为：

```ts
interface MarketQuote {
  id: 'NASDAQ' | 'SP500' | 'CSI300' | 'STAR50' | 'CSI500';
  value: number;
  change: number;
  percent: number;
  quotedAt: Date;
}
```

### 初始化集成

现有 `initClient()` 只负责发现 `marketHud` 并启动行情模块。原有 `mktBase`、随机游走和随机 `SYNC` 逻辑全部删除。

## 数据流

1. 首页加载，HUD 显示 `--` 和 `SYNCING`。
2. 浏览器立即请求 Worker 的 `/markets`。
3. Worker 检查边缘缓存；缓存未命中时才请求一次腾讯五指数合并行情。
4. Worker 安全解析 GBK 响应并返回标准 JSON。
5. 浏览器成功验证响应后，一次性更新五行数据、颜色、箭头、来源和报价时间。
6. 页面可见时，30 秒后继续刷新。
7. 页面隐藏时取消后续计时，不在后台持续请求。
8. 页面重新可见时立即刷新。
9. 成功请求会把失败退避重置为 30 秒。

## 状态与时效

行情状态分为：

- `SYNCING`：首次请求尚未完成。
- `LIVE`：当前处于对应市场的常规交易时段，且报价时间距当前不超过 10 分钟。
- `CLOSED`：当前不在对应市场的常规交易时段；显示最后报价时间。
- `STALE`：处于常规交易时段，但报价超过 10 分钟未更新。
- `OFFLINE`：首次请求失败，当前没有任何可展示的成功结果。

交易时段按工作日和时区计算：

- 中国市场：`Asia/Shanghai`，09:30–11:30、13:00–15:00。
- 美国市场：`America/New_York`，09:30–16:00；由时区转换自动处理夏令时。

本版本不内置节假日日历，因此节假日恰逢常规交易时段时，可能保守地显示 `STALE`。这比错误地显示 `LIVE` 更安全。

HUD 总状态按以下规则计算：没有任何成功数据时为 `OFFLINE`；首次请求进行中为 `SYNCING`；任一正在交易的市场数据陈旧时为 `STALE`；任一市场处于交易中且数据新鲜时为 `LIVE`；其余情况为 `CLOSED`。每行仍保留自己的报价时间和涨跌状态。

## 失败处理

- 单次请求超过 5 秒即中止。
- Worker 请求腾讯行情同样使用 5 秒超时；上游失败或响应无效时返回结构化 502。
- 连续失败后按 30 秒、60 秒、120 秒退避，最高保持 120 秒。
- 一次响应中只有部分指数有效时，只更新有效指数；缺失指数保留上次成功值并标为 `STALE`。
- 已经存在成功数据时，请求失败不清空点位，只更新状态。
- 首次请求失败时显示 `--` 和 `OFFLINE`。
- 恢复成功后立即清除错误状态并恢复 30 秒刷新。
- 页面卸载时清理计时器和未完成请求。

## 安全与资源控制

- Worker 不执行行情响应中的任何 JavaScript。
- Worker 不把远端返回的名称或 HTML 传给浏览器。
- Worker 和浏览器两层都只接受白名单中的五个代码和有限数字。
- Worker 不接受任意目标 URL，避免成为开放代理。
- CORS 响应根据固定来源白名单生成，不反射任意 `Origin`。
- 每次只发一个合并请求，禁止并发刷新。
- 页面隐藏时暂停，降低对免费数据源的请求压力。
- 页面明确标注 `DATA · TENCENT` 和“仅供展示”。

## UI 行为

- 保留现有 NERV HUD 视觉语言，不重新设计页面。
- 行顺序为 `NASDAQ`、`SP500`、`CSI300`、`STAR50`、`CSI500`。
- 涨幅为正显示绿色和 `↑`，负值显示红色和 `↓`，零值使用中性色和 `—`。
- 底部替换虚假的随机同步率，显示总体状态、数据源和最近行情时间。
- 当前移动端隐藏行情 HUD 的行为保持不变。

## 测试策略

开发遵循 RED → GREEN：

1. 为腾讯响应建立固定二进制/文本 fixture，覆盖五个指数。
2. Worker 单元测试验证字段映射、正负涨跌、GBK 字节处理、未知代码、缺字段、非法数字和部分响应。
3. Worker 单元测试验证路由、方法限制、CORS 白名单、上游 502 和缓存命中。
4. 浏览器单元测试验证 JSON 校验以及 `LIVE`、`CLOSED`、`STALE` 的边界时间。
5. 浏览器单元测试验证 30/60/120 秒退避及成功重置。
6. 结构测试确保组件只包含占位符，不再出现旧的假数据。
7. 使用 Wrangler 本地开发服务完成一次 Worker 集成测试。
8. 运行 Astro 类型检查和生产构建；已存在且与本功能无关的诊断单独记录。

行情解析和状态判断保持为纯函数，以便相关模块达到至少 80% 的分支和行覆盖率。

## 验收标准

- 源码中不存在旧的四组硬编码点位和随机行情更新逻辑。
- 首页首次加载时不会显示伪造价格。
- 正常网络下五个指数均能显示数据源返回的最新点位、涨跌幅和行情时间。
- Worker 连续请求在 20 秒内能命中边缘缓存，并返回标准 JSON 与正确 CORS 响应。
- 页面隐藏时不继续定时请求，恢复可见后立即请求。
- 请求超时、坏响应和部分响应不会产生假数据或未处理异常。
- 用户能够区分 `LIVE`、`CLOSED`、`STALE` 和 `OFFLINE`。
- 生产构建成功。

## 发布与回滚

Worker 使用 `workers/market-feed/wrangler.jsonc` 独立部署到 `workers.dev`。首次部署返回的准确 URL 将写入网站行情配置，再完成 Astro 构建、提交并推送到 `main`，由现有 GitHub Actions 发布网站到 OpenResty。

当前 Wrangler OAuth 已具备 Worker 写权限。当前 OpenResty 发布服务器的 SSH 连接故障可能阻止网站前端自动上线，但不会阻止 Worker 部署、本地实现、测试和推送。

如需回滚，只需撤销行情模块和组件改动；回滚后 HUD 应显示不可用状态，而不是恢复随机假数据。
